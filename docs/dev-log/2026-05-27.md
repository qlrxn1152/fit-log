## 발생한 문제

### 1. 문제 제목
#### 상황
- MemberRepository 테스트 코드를 작성하면서 No ParameterResolver Registered Error 

-ParameterResolutionException: No ParameterResolver registered for parameter [. . .] in constructor


#### 원인

```
@RequiredArgsConstructor
class MemberRepositoryImplTest {

    private final MemberRepository memberRepository;

```

이 코드는 오류수정전 테스트코드.

Test 는 현재, 스프링에 의존하지 않고 순수 JUnit 객체로만 작성된 상태.

애노테이션은 @RequriedArgsConstructor => 롬복은 아래와 같이 생성자를 만들어줌.

```
public MemberRepositoryImplTest(MemberRepository memberRepository) {
    this.memberRepository = memberRepository;
}
```


이후에 스프링이 생성자 1개인것을 확인하고 @Autowired 로 자동주입 시도
```
@Autowired
public MemberRepositoryImplTest(MemberRepository memberRepository) {
    this.memberRepository = memberRepository;
}
```



하지만, JUnit 입장에서는 스프링에 의존하지 않으므로, MemberRepository 를 어디서 가져와서 이 객체를 주입해줄지 전혀 모르는상태였음 즉, JUnit이 생성자 파라미터를 해결하지 못함. 

그래서 에러메세지로 나온 메세지가 

```
-ParameterResolutionException: No ParameterResolver registered for parameter [. . .] in constructor
```

즉, Test => Junit 만 실행이됐던거고, 그래서 스프링 컨테이너가 뜨지 않았고, 그래서 @Component 로 등록한 객체를 주입받을수 없는상태 또한, @Autowired 도 작동되지 않는 순수 Junit 으로 작성한 테스트코드

그래서, 

```
@Repository
@Slf4j
public class MemberRepositoryImpl implements MemberRepository {
```

테스트코드가아닌 실제 객체에서 이렇게 @Repository 를 등록했음에도 ( @Repository -> @Component 포함 ), 테스트코드에서 돌렸을때 에러가 나왔던거.

그럼 이를 어떻게 해결? 

- 순수 단위 테스트 코드를 작성
- 스프링 주입


---

#### 해결

save, FindById, FindAll 을 하는데에 굳이 Spring 을 안써도 상관없으니 순수 JUnit 으로 하기로했음.

```
class MemberRepositoryImplTest {

    private final MemberRepository memberRepository = new MemberRepositoryImpl();
```

그래서, 객체를 직접 주입해줌

---

#### 배운 점

-Test 코드는 일단 기본적으로 Spring 에 의존하지않는상태.

그래서 스프링이 필요하다면 직접 @SpringBootTest 애노테이션을 넣어줘야함.
